# Log Monitor SDK

A Spring Boot SDK for monitoring application logs and sending them to an Apache Kafka topic for centralized analysis.

This SDK provides a custom Logback appender (`KafkaAiLogAppender`) that intercepts logs and automatically forwards them to a configured Kafka topic. It is specifically optimized to only process and transmit `ERROR` level logs, minimizing performance overhead and network bandwidth.

## Features

- **Custom Logback Appender**: Intercepts `ERROR` logs directly from Logback.
- **Kafka Integration**: Asynchronously sends log events to a Kafka topic using a configured `KafkaProducer`.
- **JSON Serialization**: Automatically formats log events (including timestamps, log levels, logger names, messages, stack traces, and MDC contexts) into structured JSON.
- **Optimized Performance**:
  - Only processes `ERROR` level logs to reduce overhead.
  - Utilizes Snappy compression to reduce network bandwidth.
  - Configured with `linger.ms` (batching) for efficient throughput.

## Prerequisites

- Java 21 or higher
- Spring Boot
- Apache Maven
- Apache Kafka cluster

## Installation

Add the SDK as a dependency in your project's `pom.xml`:

```xml
<dependency>
    <groupId>in.guardianservices</groupId>
    <artifactId>log-monitor-sdk</artifactId>
    <version>0.0.1-SNAPSHOT</version>
</dependency>
```

## Configuration

To use the `KafkaAiLogAppender`, configure it in your application's `logback-spring.xml` (or `logback.xml`):

```xml
<configuration>

    <!-- Configure the custom Kafka Appender -->
    <appender name="KAFKA_APPENDER" class="in.guardianservices.log_monitor_sdk.analyzer.KafkaAiLogAppender">
        <bootstrapServers>localhost:9092</bootstrapServers> <!-- Replace with your Kafka brokers -->
        <topic>error-logs-topic</topic> <!-- Replace with your target topic -->
    </appender>

    <!-- Root logger configuration -->
    <root level="INFO">
        <appender-ref ref="KAFKA_APPENDER" />
        <!-- Add other appenders like console or file here -->
    </root>

</configuration>
```

### Properties

- `bootstrapServers`: The list of Kafka brokers (e.g., `localhost:9092,localhost:9093`).
- `topic`: The Kafka topic where error logs will be published.

## How it works

1. **Log Interception**: The `KafkaAiLogAppender` intercepts log events generated by the application using Logback.
2. **Filtering**: It ignores logs below the `ERROR` level.
3. **Mapping**: `LogEventMapper` converts the `ILoggingEvent` into a JSON string, extracting useful information like MDC (Mapped Diagnostic Context) and full stack traces.
4. **Publishing**: `KafkaLogProducer` publishes the JSON payload to the specified Kafka topic.
